# Contributing to Warehouse

Thanks for your interest in contributing! Warehouse is a code-context staging
ground for LLMs — kits, crates, pallets, and yards for assembling prompt-ready
bundles.

## Getting Started

1. Fork the repo and clone your fork.
2. Create a feature branch: `git checkout -b feat/your-change`.
3. Make your changes with focused, small commits.
4. Run tests and linting before opening a PR.
5. Open a pull request against `main` with a clear description.

## Branch Policy

- Do not commit directly to `main`.
- Use descriptive branch names: `feat/...`, `fix/...`, `chore/...`, `docs/...`.

## Pull Requests

- Keep PRs focused — one logical change per PR.
- Reference any related issues (`Fixes #123`).
- Include a test plan in the PR description.
- Ensure CI passes.

## Reporting Bugs / Requesting Features

Use the issue templates under `.github/ISSUE_TEMPLATE/`. Search existing
issues before filing a new one.
